技术架构(TA)：DeskClaw 总体技术架构视图

用户端User End

核心构成
用户门户 nodeskclaw-portal（Vue 3 + Vite + Pinia + Tailwind CSS，CE/EE 通用）；管理后台 ee/nodeskclaw-frontend（Vue 3 + shadcn-vue，仅 EE）。
主要职责
实例可视化管理、实时日志查看、拓扑图/团队协作看板展示，面向普通用户与组织管理员两类角色。

接入层Access Layer

核心构成
飞书 SSO 登录（lark-oapi）、FastAPI 路由网关、JWT 鉴权中间件、CORS 策略。
主要职责
统一登录鉴权入口，区分人类用户（JWT）与 AI 员工（proxy_token）两种身份，按角色/组织解析访问权限。

业务服务层Service Layer

核心构成
nodeskclaw-backend（Python 3.12 + FastAPI + Service Layer 模式）、nodeskclaw-llm-proxy（LLM 代理服务）、kubectl / kubernetes-asyncio（K8s 集群交互）、Alembic 迁移。
主要职责
CE/EE 双版本 FeatureGate 判断、DeskClaw 实例部署编排、集群健康巡检、Gene/技能管理、工作区任务与目标协同。

数据层Data Layer

核心构成
PostgreSQL 16（SQLAlchemy async + asyncpg）、S3/OSS 兼容对象存储（boto3）、Docker 实例数据卷 / K8s 集群资源（Pod、PVC、Deployment）。
主要职责
持久化用户/组织/实例/Gene 元数据，软删除 + Partial Unique Index 约束；实例运行时数据经 NFS/Volume 挂载与容器路径互通。
