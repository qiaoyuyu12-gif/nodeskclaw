## 1. 平台架构概览

DeskClaw（曾用名 NoDeskClaw）是一个 AI 员工实例可视化管理平台，通过 Web 界面管理 K8s / Docker 集群上的 AI 员工实例，支持一键部署、实时日志、集群健康巡检与飞书 SSO 登录。

### 1.1 版本架构

平台采用 **CE（社区版）/ EE（企业版）** 双版本架构：

| 版本 | 说明 |
|------|------|
| CE（社区版） | 本仓库开源部分，核心功能完整 |
| EE（企业版） | 私有 `ee/` 目录扩展，含多租户、高级审计、SSO 等企业特性 |

版本由 `NODESKCLAW_EDITION` 环境变量控制（`ce` / `ee`），未设置时自动检测 `ee/` 目录是否存在。

### 1.2 服务拓扑

```
                        ┌─────────────────────────────────┐
                        │          用户浏览器               │
                        └───────────────┬─────────────────┘
                                        │ HTTP/WebSocket
              ┌─────────────────────────┼───────────────────────┐
              │                         │                       │
      ┌───────▼──────┐         ┌────────▼───────┐    ┌─────────▼────────┐
      │  Portal 门户  │         │   Admin 管理后台 │    │  Backend API     │
      │  Vue 3       │         │  Vue 3 (EE)    │    │  FastAPI         │
      │  :4517 / :80 │         │  :4518 / :80   │    │  :4510 / :8000   │
      └──────────────┘         └────────────────┘    └─────────┬────────┘
                                                               │
                                              ┌────────────────┼──────────────┐
                                              │                │              │
                                    ┌─────────▼──────┐  ┌─────▼──────┐ ┌────▼────┐
                                    │  LLM Proxy     │  │ PostgreSQL │ │  K8s /  │
                                    │  FastAPI       │  │    :5432   │ │ Docker  │
                                    │  :4511 / :8080 │  └────────────┘ └─────────┘
                                    └────────────────┘
```

### 1.3 子模块说明

| 目录 | 类型 | 说明 |
|------|------|------|
| `nodeskclaw-backend/` | 后端服务 | FastAPI + SQLAlchemy，主 API 服务，端口 4510 |
| `nodeskclaw-llm-proxy/` | LLM 代理 | LLM 请求路由与鉴权代理，端口 4511 |
| `nodeskclaw-portal/` | 前端门户 | 用户门户（CE + EE），端口 4517 |
| `ee/nodeskclaw-frontend/` | 管理后台（EE） | 企业版平台管理界面，端口 4518 |
| `nodeskclaw-artifacts/` | 制品管理 | OpenClaw / Nanobot / Hermes 镜像构建脚本 |
| `openclaw-channel-nodeskclaw/` | Channel 插件 | DeskClaw channel 集成 |
| `openclaw-channel-dingtalk/` | Channel 插件 | 钉钉 Stream 协议集成 |
| `deploy/` | 部署脚本 | K8s manifest 生成与部署 CLI |
| `openclaw/src/` | 源码副本 | DeskClaw 本地副本，用于调试排查 |

## 3. 操作系统与基础环境

### 3.1 支持的操作系统

| 平台 | 支持状态 | 备注 |
|------|----------|------|
| Linux (Ubuntu 22.04+) | 完全支持 | 生产推荐环境 |
| macOS (12+) | 完全支持 | 本地开发，Apple Silicon 需注意架构差异 |
| Windows 11 (WSL2) | 支持 | 需 WSL2 环境 |
| Windows 11 (原生) | 部分支持 | 异步子进程有限制，见[常见问题](#131-windows-异步子进程错误) |

> **重要**：生产镜像目标架构为 `linux/amd64`。在 Apple Silicon (arm64) 开发机上构建 Docker 镜像时，必须指定 `--platform linux/amd64`。

### 3.2 基础工具安装

```bash
# 检查 Python 版本（需 >= 3.12）
python3 --version

# 安装 uv（Python 包管理器）
curl -LsSf https://astral.sh/uv/install.sh | sh

# 检查 Node.js 版本（需 >= 18，推荐 22）
node --version
npm --version

# 检查 Docker
docker --version
docker compose version

# 检查 kubectl（K8s 部署需要）
kubectl version --client
```

### 3.3 时区配置

平台默认使用 `Asia/Shanghai` 时区，所有服务均需设置：

```bash
# Linux
sudo timedatectl set-timezone Asia/Shanghai

# Docker 容器（通过环境变量）
TZ=Asia/Shanghai
```

- **Python**: >= 3.12
- **包管理器**: `uv`（推荐，替代 pip/poetry）

### 4.2 后端依赖（nodeskclaw-backend）

```bash
cd nodeskclaw-backend
uv sync          # 安装全部依赖（含开发依赖）
uv sync --no-dev # 生产环境（仅运行时依赖）
```

| `alembic` | >= 1.13.0 | 数据库迁移 |

| `httpx` | >= 0.28.0 | 异步 HTTP 客户端 |
| `kubernetes-asyncio` | >= 31.1.0 | K8s 异步 SDK |
| `lark-oapi` | >= 1.5.3 | 飞书 OpenAPI SDK |

| `pytest` | >= 8.0.0 |
| `pytest-asyncio` | >= 0.24.0 |
| `ruff` | >= 0.8.0 |

### 4.3 LLM Proxy 依赖（nodeskclaw-llm-proxy）

```bash
cd nodeskclaw-llm-proxy
uv sync
```

| 包名 | 版本要求 | 说明 |
|------|----------|------|

| `httpx` | >= 0.28.0 | 请求代理 |

> LLM Proxy 容器内额外需要 **Node.js 22 + npm** 用于 OpenAI Codex CLI（版本 0.116.0）。

### 4.4 常用开发命令

```bash
# 代码检查
uv run ruff check .

# 自动修复
uv run ruff check --fix .

# 运行测试
uv run pytest

# 运行单个测试
uv run pytest app/services/test_xxx.py::test_foo

# 数据库迁移
uv run alembic upgrade head

# 生成迁移文件（修改 Model 后必须执行）
uv run alembic revision --autogenerate -m "描述变更内容"
```

## 5. Docker 容器化部署

### 5.1 镜像架构

| 镜像 | 基础镜像 | 目标架构 |
|------|----------|----------|
| `nodeskclaw-backend` | `python:3.12-slim` | `linux/amd64` |
| `nodeskclaw-llm-proxy` | `python:3.12-slim` | `linux/amd64` |
| `nodeskclaw-portal` | `node:22-alpine` → `nginx:1.27-alpine` | `linux/amd64` |
| `nodeskclaw-admin` (EE) | `node:22-alpine` → `nginx:1.27-alpine` | `linux/amd64` |

> **注意**：在 Apple Silicon Mac 上构建时，必须使用 `--platform linux/amd64` 参数。

### 5.2 Docker Compose 快速启动

```bash
# CE 模式（默认）
docker compose up -d

# EE 模式
docker compose -f docker-compose.yml -f docker-compose.ee.yml up -d

# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f nodeskclaw-backend

# 停止服务
docker compose down
```

### 5.3 docker-compose.yml 服务说明

| 服务名 | 镜像 | 端口映射 | 重启策略 |
|--------|------|----------|----------|
| `postgres` | `postgres:16-alpine` | `5432:5432` | `unless-stopped` |
| `nodeskclaw-backend` | `nodeskclaw-backend:${DESKCLAW_VERSION}` | `4510:8000` | `unless-stopped` |
| `llm-proxy` | `nodeskclaw-llm-proxy:${DESKCLAW_VERSION}` | `4511:8080` | `unless-stopped` |
| `portal` | `nodeskclaw-portal:${DESKCLAW_VERSION}` | `80:80` | `unless-stopped` |
| `nodeskclaw-admin` (EE) | `nodeskclaw-admin:${DESKCLAW_VERSION}` | `8001:80` | `unless-stopped` |

**Volume 挂载：**

| 服务 | 挂载路径 | 说明 |
|------|----------|------|
| postgres | `pg_data:/var/lib/postgresql/data` | 数据库持久化 |
| nodeskclaw-backend | `/var/run/docker.sock:/var/run/docker.sock` | Docker Socket（管理容器） |
| nodeskclaw-backend | `nodeskclaw-data:/nodeskclaw-data` | 实例数据目录 |
| llm-proxy | `$HOME/.codex:/root/.codex` | Codex CLI 缓存 |

### 5.4 关键环境变量（Docker Compose）

```dotenv
# 版本标签
DESKCLAW_VERSION=v0.8.3

# 数据目录（Windows 必须显式设置）
NODESKCLAW_DATA_DIR=${HOME}/.nodeskclaw/docker-instances
# Windows 示例: NODESKCLAW_DATA_DIR=C:\Users\username\.nodeskclaw\docker-instances

# 版本模式
NODESKCLAW_EDITION=ce   # ce 或 ee

# 时区
TZ=Asia/Shanghai
```

### 5.5 镜像私有仓库镜像（离线/内网环境）

通过环境变量配置镜像源，无需修改 Dockerfile：

```dotenv
PIP_INDEX_URL=https://mirrors.aliyun.com/pypi/simple/

APT_MIRROR=mirrors.aliyun.com

```

### 5.6 健康检查

```bash

# 检查后端 API
curl http://localhost:4510/health
```

## 6. 数据库与存储配置

### 6.1 PostgreSQL 配置

**版本要求**：PostgreSQL >= 16

**连接字符串格式：**
```
postgresql+asyncpg://<user>:<password>@<host>:<port>/<database>
```

**默认配置：**

| 参数 | 开发环境 | Docker Compose |
|------|----------|----------------|
| Host | `localhost` | `postgres`（服务名） |
| Port | `5432` | `5432` |
| User | `nodeskclaw` | `nodeskclaw` |
| Password | `nodeskclaw123` | `nodeskclaw` |
| Database | `nodeskclaw_rbac` | `nodeskclaw_rbac` |

**连接池配置：**

```dotenv
DB_POOL_SIZE=10           # 连接池大小
DB_POOL_MAX_OVERFLOW=20   # 最大溢出连接数
```

**多数据库后缀（可选）：**

```dotenv
DATABASE_NAME_SUFFIX=dev   # 数据库名变为 nodeskclaw_rbac_dev
# 留空则使用 DATABASE_URL 原始库名
# 设为 auto 则使用 hostname 作为后缀
```

### 6.2 数据库迁移

```bash
cd nodeskclaw-backend

# 应用全部迁移（容器启动时自动执行）
uv run alembic upgrade head

# 查看当前版本
uv run alembic current

# 查看迁移历史
uv run alembic history

# 回滚一步
uv run alembic downgrade -1

# 新增 Model 后生成迁移
uv run alembic revision --autogenerate -m "add_xxx_table"
```

> **规范**：禁止手写 `revision ID`，必须通过 `--autogenerate` 自动生成。

### 6.3 数据库备份与恢复

```bash
# 备份（Docker Compose 环境）
docker compose exec postgres pg_dump -U nodeskclaw nodeskclaw_rbac > backup_$(date +%Y%m%d).sql

# 恢复
docker compose exec -T postgres psql -U nodeskclaw nodeskclaw_rbac < backup_20240101.sql
```

### 6.4 对象存储（可选，S3 兼容）

支持接入任意 S3 兼容对象存储（阿里云 OSS、MinIO 等）：

```dotenv
S3_ENDPOINT=https://oss-cn-hangzhou.aliyuncs.com
S3_REGION=cn-hangzhou
S3_BUCKET=my-nodeskclaw-bucket
S3_ACCESS_KEY_ID=your-access-key
S3_SECRET_ACCESS_KEY=your-secret-key
S3_KEY_PREFIX=prod
```

未配置时，文件存储至本地目录：

```dotenv
LOCAL_STORAGE_DIR=/data/nodeskclaw-files
```

## 7. 服务部署

### 7.1 本地开发一键启动

```bash
# 自动检测版本（推荐）
./dev.sh

# 强制 CE 模式
./dev.sh ce

# 强制 EE 模式
./dev.sh ee

# 清洁启动（删除缓存，重新安装依赖）
./dev.sh --fresh

# 使用 Docker 运行 PostgreSQL
./dev.sh --docker-pg
```

**启动后访问地址：**

| 服务 | 地址 |
|------|------|
| Portal 门户 | http://localhost:4517 |
| Admin 管理后台 (EE) | http://localhost:4518 |
| Backend API | http://localhost:4510 |
| API 文档 | http://localhost:4510/docs |
| LLM Proxy | http://localhost:4511 |

### 7.2 手动启动各服务

```bash
# 后端
cd nodeskclaw-backend
uv run uvicorn app.main:app --reload --port 4510

# LLM Proxy
cd nodeskclaw-llm-proxy
uv run uvicorn app.main:app --reload --port 4511

# Portal 前端
cd nodeskclaw-portal
npm install && npm run dev

# Admin 前端（EE）
cd ee/nodeskclaw-frontend
npm install && npm run dev
```

### 7.3 前端构建

```bash
# Portal
cd nodeskclaw-portal
npm run build    # 产物输出至 dist/

# Admin（EE）
cd ee/nodeskclaw-frontend
npm run build
```

### 7.4 K8s 集群部署

> **重要**：部署脚本必须由用户手动执行，禁止 AI 自动运行 `deploy/cli.sh`。

```bash
# CE 模式部署
./deploy/cli.sh --context <k8s-context-name>

# EE 模式部署
./deploy/cli.sh --context <k8s-context-name> --ee

# 查看部署状态
kubectl get pods -n <namespace> --context <context-name>

# 查看 Pod 详情
kubectl describe pod <pod-name> -n <namespace> --context <context-name>

# 查看日志
kubectl logs <pod-name> -n <namespace> --context <context-name> --tail=100
```

> **强制规则**：所有 kubectl 命令必须显式指定 `--context <name>`，禁止依赖 current-context 默认值。

## 9. 认证与授权配置

### 9.1 JWT 配置

```dotenv
JWT_SECRET=your-super-secret-key-change-in-production   # 生产必须更换
JWT_ALGORITHM=HS256
JWT_EXPIRE_HOURS=24    # Token 有效期（小时）
```

> **安全要求**：`JWT_SECRET` 在生产环境必须使用强随机字符串，建议 64 字符以上。

```bash
# 生成安全随机密钥
openssl rand -hex 32
```

### 9.2 数据加密配置

平台使用 AES-256-GCM 对 KubeConfig 等敏感数据加密存储：

```dotenv
ENCRYPTION_KEY=change-me-32-bytes-base64-key__=   # 32 字节 Base64 编码密钥
```

```bash
# 生成 32 字节 Base64 密钥
python3 -c "import secrets, base64; print(base64.b64encode(secrets.token_bytes(32)).decode())"
```

### 9.3 飞书 SSO 配置

**Admin 管理后台应用（EE）：**

```dotenv
FEISHU_APP_ID=cli_xxxxxxxxxxxx
FEISHU_APP_SECRET=your-app-secret
FEISHU_REDIRECT_URI=https://your-domain.com/login/callback/feishu
```

**Portal 门户应用（可选）：**

```dotenv
FEISHU_APP_ID_PORTAL=cli_xxxxxxxxxxxx
FEISHU_APP_SECRET_PORTAL=your-portal-app-secret
```

**飞书应用配置步骤：**

1. 登录[飞书开放平台](https://open.feishu.cn)，创建企业自建应用
2. 开启「网页应用」，配置重定向 URL 为 `FEISHU_REDIRECT_URI` 的值
3. 申请权限：`contact:user.base:readonly`（获取用户信息）
4. 发布应用并获取 `App ID` 与 `App Secret`

### 9.4 管理员账号初始化

## 13. 常见问题与排错

### 13.1 Windows 异步子进程错误

**现象**：`cluster_service._create_docker_cluster` 报错，前端显示「连接失败:」后空白。

**原因**：`asyncio.create_subprocess_exec` 在 Windows SelectorEventLoop 上抛出 `NotImplementedError`，被通用 `except Exception` 吞没，`str(NotImplementedError())` 为空字符串。

**解决**：

```python
try:
    proc = await asyncio.create_subprocess_exec(...)
except NotImplementedError:
    # Windows SelectorEventLoop 回退到同步路径
    result = await asyncio.to_thread(subprocess.run, ...)
```

### 13.2 数据库迁移冲突

**现象**：`alembic upgrade head` 报 `Multiple heads`。

**解决**：

```bash
# 查看分支头
uv run alembic heads

# 合并分支
uv run alembic merge heads -m "merge_migrations"

# 再次升级
uv run alembic upgrade head
```

### 13.3 Gene 表按 slug 查询返回多行

**现象**：接口 500，报错 `MultipleResultsFound`。

**原因**：fork 三向架构后，同 slug 可在多 scope 并存，DB partial unique 是 `(slug, org_id)` 而非全局。

**解决**：
- 列表/兜底：使用 `.scalars().first()` 而非 `scalar_one_or_none()`
- 精确定位：使用 `gene_service.get_gene_by_slug_in_scope(slug, org_id, created_by)`

### 13.4 Docker 镜像架构不匹配

**现象**：Apple Silicon Mac 上构建的镜像在 amd64 集群运行报错。

**解决**：始终指定目标平台：

```bash
docker build --platform linux/amd64 -t nodeskclaw-backend:latest .
```

### 13.5 飞书 SSO 回调失败

**现象**：飞书登录后重定向报 404 或 CSRF 错误。

**排查步骤**：
1. 确认飞书应用后台已添加重定向 URI（与 `FEISHU_REDIRECT_URI` 完全一致）
2. 确认 `FEISHU_APP_ID` 和 `FEISHU_APP_SECRET` 填写正确
3. 检查 `CORS_ORIGINS` 包含前端域名

### 13.6 端口占用

**现象**：`dev.sh` 启动报端口已被占用。

```bash
# 查找占用 4510 端口的进程
# Linux/macOS
lsof -i :4510

# Windows
netstat -ano | findstr :4510

# 终止进程（Linux/macOS）
kill -9 <PID>
```

### 13.7 数据库连接失败

**现象**：后端启动报 `asyncpg.exceptions.ConnectionRefusedError`。

**排查**：

```bash
# 检查 PostgreSQL 是否运行
docker compose ps postgres

# 检查连接
docker compose exec postgres psql -U nodeskclaw -c "\l"

# 重启 PostgreSQL
docker compose restart postgres
```

### 13.8 前端依赖安装失败（npm）

**国内网络问题解决：**

```bash
# 切换淘宝镜像
npm config set registry https://registry.npmmirror.com

# 清除缓存后重试
npm cache clean --force
npm install
```
