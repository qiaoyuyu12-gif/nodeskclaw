3 种集成架构方案

选择最符合你团队技术路线和维护成本的方案

A

RAGFlow 直连网关模式 推荐

后端新增 skill 模块直接调用 RAGFlow REST API；Gene 技能包新增 rag_query 能力类型；Portal 新增"技能库"页面。RAGFlow 凭证由管理员在 Admin 后台统一配置。

优点

  • 改动范围最小，复用现有 Gene/Channel 体系
  • RAGFlow 升级不影响核心代码
  • 员工通过现有 Portal 无缝使用

缺点

  • RAGFlow 不可用时技能降级需要手动处理
  • 跨服务调试链路略长
B

LLM Proxy 统一代理模式

将 RAGFlow 作为一种"知识增强 Provider"注册到现有 nodeskclaw-llm-proxy；Agent 发送请求时 Proxy 自动注入 RAG 上下文，对 Gene 透明。

优点

  • 现有 Gene 无需改造，RAG 能力对 Agent 透明
  • 复用 Proxy 的 token 统计和配额体系

缺点

  • RAG 注入时机和策略难以精细控制
  • Proxy 变成单点，职责过重
  • 员工无法主动选择使用哪个知识库
C

知识库 Gene 插件模式

将每个 RAGFlow 知识库封装为独立 Gene（ragflow-kb-{name}），通过 GeneHub 分发；Agent 按需 install Gene 后即可调用对应知识库。

优点

  • 知识库即技能，与 Gene 生态完美融合
  • 细粒度权限：不同 Agent 绑定不同知识库

缺点

  • Gene 数量随知识库增长爆炸式增长
  • 知识库更新需重新打包 Gene，运维成本高
